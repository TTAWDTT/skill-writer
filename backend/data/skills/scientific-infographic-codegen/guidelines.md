# scientific-infographic-codegen Guidelines

目标：在**非成图模型**条件下，用 LLM 先做信息压缩，再输出可本地渲染的信息图 JSON。

## 工作流（必须遵守）

1. **先概括**：从“重点片段 + 全文上下文”提炼 3-6 条 `concise_summary`。
2. **再结构化**：基于摘要生成 `diagram_spec`（严格匹配 schema）。
3. **最后自检**：检查重复、长度、术语一致性，再输出。

## 硬约束

1. 只输出 JSON，不输出解释、Markdown、代码块标记。
2. 重点片段优先，全文用于约束术语和逻辑一致性。
3. 禁止复制长段落；bullets 必须改写成短语要点。
4. 禁止阶段重复（同义复述也算重复）。
5. 不得编造数据、实验结果、指标结论。
6. 仅生成本地渲染能力范围内的结构（节点/层次/流程），不要求照片级视觉。

## 结构语义建议（参考 scientific-schematics）

- 技术路线：问题定义 → 方法实现 → 验证评估 → 成果输出。
- 研究框架：目标 → 科学问题/假设 → WP 任务包 → 支撑条件 → 预期成果。
- 节点名建议 4-12 字，单节点要点 2-4 条，30 秒可读。

## 文本规范

- 每条 bullets 建议 8-20 字。
- 用“动作 + 对象”或名词短语表达。
- 术语口径前后一致，避免同义词混写。
