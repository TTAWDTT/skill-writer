## 结构质量
- JSON 可直接解析。
- 顶层包含 `concise_summary` 与 `diagram_spec`（允许后端兼容旧格式）。
- `diagram_spec` 与 schema 一致，关键字段不缺失。

## 信息质量
- 与全文和重点片段一致，不引入外部事实。
- 要点覆盖核心逻辑链路（目标/问题-方法-验证-产出）。
- 术语口径统一，不出现同义词混写。

## 冗余控制
- 不直接复制长段落。
- 同一要点不在多个阶段/模块重复。
- bullets 精炼（建议 8-20 字）。

## 可视化友好
- 标题短而可辨识。
- 单节点信息量适中（2-4 条要点）。
- 层次关系清晰，适合本地渲染布局。
